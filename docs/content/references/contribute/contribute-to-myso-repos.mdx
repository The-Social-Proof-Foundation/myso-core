This page describes how to contribute toMySo, and provides additional information about participating in the MySo community.

## Join the community {#join-the-community}

To connect with the MySo community, join our [Discord](https://discord.gg/myso).

## Open issues {#open-issues}

To report an issue withMySo, [create an issue](https://github.com/the-social-proof-foundation/myso-core/issues/new/choose) in the GitHub repo. Click **Get started** to open a template for the type of issue to create.

## Fork MySo to contribute {#fork-myso-contribute}

To contribute to MySo source code or documentation, you need only a GitHub account. You can commit updates and then submit a PR directly from the Github website, or create a fork of the repo to your local environment and use your favorite tools to make changes. Always submit PRs to the `main` branch.

See [MySo Environment Setup](./myso-environment.mdx) for instructions on forking the MySo repository, if necessary.

## Contribute via the MySo Improvement Proposal (SIP) process {#SIP}

The MySo Network is an open source, decentralized, and permissionless protocol that welcomes community contributions. If you have an idea regarding a core protocol upgrade, the best way to make your voice heard is via submitting a MySo Improvement Proposal (SIP). For more information on SIPs, see [Contribute to MySo through SIPs](https://blog.mysocial.network/myso-improvement-proposals-sips/).
